Vuetify 4 テーマの仕組み

テーマ定義 → Vuetify が CSS 変数を自動生成

// src/plugins/vuetify.ts
createVuetify({
  theme: {
    defaultTheme: 'dark',
    themes: {
      dark:     darkTheme,
      light:    lightTheme,
      practice: practiceTheme,
    },
  },
})

↓ Vuetify が自動で CSS 変数を注入

/* ブラウザで実際に生成される変数（例） */
--v-theme-primary:   33,150,243;   /* #2196F3 を RGB 分解 */
--v-theme-secondary: 144,202,249;
--v-theme-background: 18,18,18;

color="primary" は内部で background-color: rgb(var(--v-theme-primary)) を参照する

このプロジェクトの 3 テーマ

// src/plugins/vuetify.ts（実コード抜粋）
const darkTheme: ThemeDefinition = {      // ダーク（デフォルト）
  dark: true,
  colors: { primary: '#2196F3', background: '#121212', surface: '#1E1E1E', ... },
}
const lightTheme: ThemeDefinition = {     // ライト（薄青基調）
  dark: false,
  colors: { primary: '#1565C0', background: '#EFF7FF', surface: '#FFFFFF', ... },
}
const practiceTheme: ThemeDefinition = {  // プラクティス（オレンジ基調）
  dark: false,
  colors: { primary: '#E65100', background: '#FFF3E0', surface: '#FFFFFF', ... },
}

切り替え方法：useTheme().global.name.value = 'light'

// src/stores/theme.ts — Pinia ストア経由で管理
export const useThemeStore = defineStore('theme', () => {
  const currentTheme = ref<AppTheme>(
    (localStorage.getItem('appTheme') as AppTheme) ?? 'dark'
  )
  function setTheme(theme: AppTheme) {
    currentTheme.value = theme
    localStorage.setItem('appTheme', theme)
  }
  return { currentTheme, setTheme }
})

App.vue: <v-app :theme="themeStore.currentTheme"> で全体に適用

色設定の粒度（4 段階）

粒度手段例
アプリ全体 <v-app :theme="name"> themeStore で切り替え
ページ・セクション単位 theme prop <MainLayout theme="light">
部品 1 つ color prop 直値 <v-btn color="#E65100">
CSS 変数直書き :style :style="{'--v-theme-primary': '255,0,0'}"

通常はアプリ全体 or ページ単位で十分。部品レベルは例外的な固定色のみ。

セマンティックカラー vs 直値

✅ テーマ追従（推奨）

<v-btn color="primary">送信</v-btn>
<v-card color="surface">...</v-card>
<v-chip color="warning">注意</v-chip>

テーマ切り替えで色が一括変わる

⚠️ 固定（例外的に使用）

<v-btn color="#E65100">送信</v-btn>
<v-card color="#FFFFFF">...</v-card>

テーマが変わっても色が変わらない

判断基準：
テーマで一括管理したい → セマンティック名（primary 等）
ブランドカラー等で固定したい → 直値（#E65100）

カスタムセマンティックカラーの追加

ThemeDefinition.colors には [key: string]: Color の index signature があるため、任意のキーを追加できる

// src/plugins/vuetify.ts に追加
const darkTheme: ThemeDefinition = {
  dark: true,
  colors: {
    primary:    '#2196F3',
    // ... 既存 ...
    'brand-accent': '#FF6F00',   // カスタムセマンティックカラー
    'status-ok':    '#00C853',
  },
}
<!-- テンプレートでセマンティック名で参照 -->
<v-card color="brand-accent">ブランド強調カード</v-card>
<v-chip color="status-ok">正常</v-chip>
/* Vuetify が自動生成する CSS 変数 */
color: rgb(var(--v-theme-brand-accent));
color: rgb(var(--v-theme-status-ok));

テーマで制御できる範囲（実測）

項目制御できるか手段
罫線の透明度 ✅ variables['border-opacity']（デフォルト 0.12）
罫線の色 ⚠️ 間接的のみ on-surface カラーを変える
文字の色 ✅ on-surface / on-background
文字の強調度 ✅ high-emphasis-opacity（dark: 0.87 / light: 1.0）
medium-emphasis-opacity（dark: 0.60 / light: 0.70）
フォントファミリー ✅ --v-font-body / --v-font-heading
フォントサイズ ❌ Vuetify 管轄外 グローバル CSS 上書きが必要（.text-h1 等は固定）
カスタム変数 ✅ 自由に追加 variables に任意キー

まとめ・チーム規約

  • Vuetify 4 のテーマは CSS 変数 に変換されて自動適用される
  • このプロジェクトは 3 テーマ（dark / light / practice）を Pinia ストア + localStorage で切り替え
  • 色の指定は セマンティック名優先（primary, surface 等）
  • チーム独自の色は colors に カスタムセマンティックカラー として追加
  • variables で罫線・強調度・フォントを制御可能
  • フォントサイズは Vuetify の管轄外 — グローバル CSS で対応
チーム規約：
「色は直書きせず color="primary" 等セマンティック名を使う。
固定したいときだけ直値（#E65100）を許容する。」
