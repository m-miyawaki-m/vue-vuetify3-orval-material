原則：データは props で上から下へ、イベントは emit で下から上へ。 子コンポーネントは「表示と操作の検知」だけを担い、ビジネスロジックは親が持つ。
部品化すべき？ 判断基準
✅ コンポーネントに切り出す 同じ見た目が 2か所以上 loading・disabled など 内部状態 を持つ スタイルを カプセル化 したい 単体テスト を書きたい 🔲 Vuetify をそのまま使う 1か所のみ、再利用の予定なし Vuetify の props だけで要件が満たせる 独自ロジックや状態が不要 ラッパーが 薄すぎてメリットがない
