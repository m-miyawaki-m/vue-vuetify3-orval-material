設計提案 ─ Section 3：ルーター構成と画面一覧

各画面がどのレイアウトを使い、BottomNavのどのタブに対応するかを示します。

ルート定義
// router/index.ts
// ── MainLayout タブ（BottomNav に対応）──
{ path: '/', component: HomePage, tab: 'home' }
{ path: '/search', component: SearchPage, tab: 'search' }
{ path: '/favorites', component: FavoritePage, tab: 'favorites' }
{ path: '/settings', component: SettingsPage, tab: 'settings' }
// ── SubLayout スタック遷移 ──
{ path: '/detail/:id', component: DetailPage, props: true }

画面一覧と使用レイアウト

画面 パス レイアウト BottomNavタブ 状態
HomePage / MainLayout 🏠 ホーム 既存 MenuPage を改修
SearchPage /search MainLayout 🔍 検索 既存を改修
FavoritePage /favorites MainLayout ❤ お気に入り 新規（プレースホルダー）
SettingsPage /settings MainLayout ⚙ 設定 新規（プレースホルダー）
DetailPage /detail/:id SubLayout なし（スタック遷移） 既存を改修

BottomNav のタブ定義

// MainLayout 内部で定義

{ icon: 'mdi-heart', label: 'お気に入り', to: '/favorites' }
{ icon: 'mdi-cog', label: '設定', to: '/settings' }
