レイアウト実装アプローチ：3案

同じ見た目でも、コンポーネントの構造が違います。保守性とシンプルさのバランスで選んでください。

MainLayout.vue
AppBar (タイトル・アイコン)
<slot /> ← ページコンテンツ
BottomNav (4タブ)
SubLayout.vue
AppBar (← 戻る・タイトル)
<slot /> ← ページコンテンツ
BottomNav なし

A: 2種レイアウト（推奨）

MainLayout（タブ画面用）と SubLayout（詳細画面用）の2コンポーネント。各ページが明示的にどちらを使うか選ぶ。構造が明確で、将来の変更が容易。

メリット

  • 役割が明確
  • DetailPage の拡張が簡単
  • テンプレートとして再利用しやすい

デメリット

  • ファイルが2つ増える
AppLayout.vue (1つだけ)
AppBar
<slot />
BottomNav v-if="showNav"
← showNav は route.meta で切り替え

B: 単一レイアウト（props切り替え）

1つの AppLayout が props や route.meta で「BottomNavを出すか」「戻るボタンを出すか」を制御。ファイルは少ないが props が増えて複雑になりやすい。

メリット

  • ファイルが1つで済む

デメリット

  • props / meta が増えると管理しづらい
  • 条件分岐が積み重なりやすい
App.vue（全部ここに）
AppBar
<router-view />
BottomNav
← 各ページはコンテンツのみ
← 詳細画面の戻るボタンはどこに？

C: App.vue 集中管理

AppBar と BottomNav を App.vue に置き、router-view でページ内容だけ切り替え。一見シンプルだが、DetailPage の「戻るボタン付きヘッダー」など画面ごとの差異を App.vue で吸収する必要があり複雑化しやすい。

メリット

  • 構成ファイルが最小

デメリット

  • ページ固有の差異が App.vue に集まる
  • 拡張するほど管理が難しくなる
